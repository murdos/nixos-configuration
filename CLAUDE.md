# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Key Commands

```bash
# Apply configuration to the running system
sudo nixos-rebuild switch

# Build without activating (dry run / sanity check)
sudo nixos-rebuild build

# Show what packages changed between generations (uses nvd, installed in system-packages.nix)
nvd diff /nix/var/nix/profiles/system-{N-1,N}-link

# Garbage-collect old generations
sudo nix-collect-garbage -d

# nh (nix helper, enabled in system-packages.nix) — friendlier wrapper
nh os switch
nh os build
```

## Architecture

`configuration.nix` is a **symlink** pointing to the active machine file (e.g. `machines/framework-amd-7040.nix`). Switching machines means re-pointing the symlink.

### Module hierarchy

```
machines/<machine>.nix        ← entry point per host; imports:
  commons.nix                 ← always imported; pulls in:
    acpi.nix / fonts.nix / network.nix / users.nix / system-packages.nix
  hardware-configuration.nix  ← generated by nixos-generate-configuration
  desktop-{gnome,hyprland,xfce}.nix  ← mutually exclusive desktop choice; each imports:
    desktop-base.nix          ← shared X/Wayland, Pipewire, input
  network-shares.nix          ← NFS mounts
  virtualisation.nix          ← Docker + VirtualBox
```

Machines also directly import from **nixos-hardware** (via `builtins.fetchGit` or channel path) for hardware-specific quirks.

### Adding a new machine

1. Copy the closest existing `machines/*.nix` as a starting point.
2. Run `nixos-generate-configuration` to regenerate `hardware-configuration.nix`.
3. Symlink `configuration.nix` → the new machine file.
4. Choose a desktop by importing exactly one of `desktop-gnome.nix`, `desktop-hyprland.nix`, or `desktop-xfce.nix`.

### Desktop environments

- **GNOME** (`desktop-gnome.nix`): default for all current machines; adds GNOME Keyring, browser connector, Plotinus.
- **Hyprland** (`desktop-hyprland.nix`): Wayland compositor with UWSM; includes waybar, hyprpaper, hyprlauncher.
- **XFCE** (`desktop-xfce.nix`): lightweight alternative.

### Notable global settings (commons.nix)

- Locale: `en_US.UTF-8`, keyboard `fr`, timezone `Europe/Paris`.
- Nix: flakes + nix-command enabled, auto-optimise store, weekly GC (delete >20 days), daily auto-upgrade.
- `nix-ld` enabled (run unpatched dynamic binaries).
- `allowUnfree = true`.

### User

Single user `amino` (uid 1000), shell `fish`, groups include `wheel`, `docker`, `networkmanager`. `mutableUsers = true` — password changes survive rebuilds.

### Secrets

`age` and `sops` are installed in `system-packages.nix` for secret management (no sops-nix integration yet — secrets workflow is manual).
